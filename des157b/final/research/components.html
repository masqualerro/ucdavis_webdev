Features

  • Ranking
    • User will have the capability to rank pieces of art
    • User will drag and drop each piece of art into order from best to worst
  • Submitting Data
    • User will browse through gallery of random art pieces, and submit their reaction to it (All of their submitted data will then be displayed and visualized)
  • Music Player
    • User will cycle through a list of curated songs, and react to them, after which data will be displayed and visualized
  • Random art display
    • User can cycle through a gallery of randomly selected fine art pieces
    • This will not be the main feature of the website, just a fun side-feature
    • I plan to use the Chicago Art Institute API for this

Libraries & APIs

  • Art Institute of Chicago API
    • I plan to use this API to display random art pieces, and create enough variation in ranking/comparing to compel the user to revisit the website.
    • According to the documentation: The Art Institute of Chicago's API provides JSON-formatted data as a REST-style service that allows developers to explore and integrate the museum’s public data into their projects.
    • I have become familiar with JSON in my “Check Yourself” and “Library Liberty” projects, so I feel confident that I will be able to work with the API in this regard.
    • I am unfamiliar with the GET and POST queries. e. I plan on following the quick start guide on the API website to learn how to use this.
  • Chart.js OR D3.js
    • Data visualization will be a large part of my website, and I will use either D3.js or Chart.js to achieve this
    • I am comfortable with Chart.js from my Library Liberty project
    • D3.js is far more complex than Chart.js, and I will have to watch tutorials/learn it from scratch
    • D3.js is more complex and vibrant than Chart.js, so I am leaning towards it for the data visualization in my project.
